# RestroSuite Client Demo Script

Use this flow for restaurant demos. Keep the demo practical and operations-focused.

## Demo Setup

- Use a clean demo tenant.
- Add 5 menu items.
- Add inventory for 3 ingredients.
- Add 2 staff users.
- Keep one sample QR order ready.
- Keep Growth Hub empty enough to show creation.

## 12-Minute Demo Flow

1. Open dashboard and show the app-like mobile-ready layout.
2. Add an item to cart and create a POS bill.
3. Show bill history and export.
4. Show QR ordering from customer view.
5. Accept the QR order and open KDS.
6. Show inventory and low-stock thinking.
7. Show staff access controls.
8. Open Growth Hub:
   - create support ticket
   - add reservation
   - add purchase order
   - add item costing
   - create offer
   - create refund request
9. Show backup export.
10. Close with zero-cost pilot and upgrade path.

## Questions to Ask

- How do you currently manage bills and QR orders?
- Do you need staff permissions?
- Do you use reservations or waitlists?
- Do you track ingredient cost?
- Do you want Android app access?
- What would block you from using this tomorrow?

## Close

Offer a pilot setup:

- one outlet
- limited staff
- guided menu setup
- one-week support window
- upgrade only after value is proven
